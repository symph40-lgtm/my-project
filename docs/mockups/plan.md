# Mockup Plan

## Overview

마켓가드 프론트엔드 목업 스프린트. 직장인 개인투자자가 아침·장중·마감 전 3개 시간대에 걸쳐 시장 위험도를 확인하고 방어 행동 기준을 세우는 핵심 흐름을 화면 단위로 설계한다.

코드 없이 화면 구조와 정보 우선순위를 사용자와 리뷰할 수 있는 목업 계획 문서다.

## Source Context

- PRD: 3개 User Scenarios (아침 방어 브리핑 / 장중 판단 보정 / 마감 전 리스크 정리), FR-001~018, 한국장 영향도(위험/중립/우호) + 근거 + 행동 가이드가 핵심 가치
- User Flows: 플로우 1(아침 5단계) · 플로우 2(장중 5단계) · 플로우 3(마감 전 5단계), 화면 인벤토리 8개
- Additional Context: DESIGN.md (Guard Mint `#04c584`, 2px radius, Pretendard 700, 카드 밀도 우선, 시멘틱 색상 위험=Error Red / 경고=Warning Orange)

## Mockup Scope

- 전체 3개 플로우에 걸쳐 사용자가 실제로 보고 행동하는 8개 화면
- 각 화면의 기본(default) 상태 + 빈 상태·로딩·오류 상태 포함
- 모바일 우선 레이아웃 (375px 기준), 데스크톱 확장 방향 명시
- 더미 데이터로 동작 (실제 API 연동 없음)

## Out of Scope

- 실제 금융 데이터 API 연동 및 LLM 브리핑 생성
- 사용자 계정·로그인·인증 흐름
- 증권사 계좌 자동 연동
- 웹 푸시 알림 실제 구독 로직
- 매수·매도 실행 기능
- 과거 유사 구간 아카이브

## Screen Flow

```text
[S001 오늘의 방어 브리핑]
  --> [S002 근거 상세]
        --> [S004 민감도 진단]
              --> [S005 행동 가이드]
  --> [S003 내 포트폴리오 입력]
        --> [S004 민감도 진단]
              --> [S005 행동 가이드]

[S006 장중 변화 알림]  ← 알림/장중 재진입
  --> [S001 오늘의 방어 브리핑 (장중 탭)]
        --> [S002 근거 상세]
              --> [S004 민감도 진단]
                    --> [S005 행동 가이드]

[S001 오늘의 방어 브리핑 (마감 전 탭)]
  --> [S007 마감 전 점검]
        --> [S004 민감도 진단]
        --> [S002 근거 상세 (내일 이벤트)]
        --> [S005 행동 가이드 (내일 기준)]
              --> [S008 아침 알림 설정]
```

## [ ] Screen S001 - 오늘의 방어 브리핑

**Purpose:** 사용자가 앱을 열자마자 오늘의 한국장 영향도와 핵심 근거 3개를 즉시 확인한다. 시간대 탭(아침/장중/마감 전)으로 현재 시간대 브리핑에 바로 진입.

**User Moment:** 플로우 1 단계 1 — "앱을 열고 오늘의 영향도 확인", 플로우 2 단계 2 — "아침 판단이 유지되는지 확인", 플로우 3 단계 1 진입점

**Primary Action:** [근거 보기] 또는 [내 포트 점검] 선택

**Leads To:** S002 (근거 보기), S003 (포트 점검), S007 (마감 전 탭)

**States to Mock:** 기본(위험), 중립, 우호, 로딩(데이터 수신 중), 오류(데이터 fetch 실패), 포트폴리오 미입력(빈 포트 상태)

**Review Focus:**
- [ ] 영향도(위험/중립/우호)가 첫눈에 들어오는가?
- [ ] 시간대 탭 전환이 직관적인가?
- [ ] 포트폴리오 미입력 상태에서도 브리핑 가치가 전달되는가?

**Done When:**
- [ ] 위험/중립/우호 3가지 영향도 상태가 모두 시각화됨
- [ ] 시간대 탭(아침/장중/마감 전) 레이아웃이 표현됨
- [ ] 로딩·오류·빈 포트폴리오 상태가 표현됨

## [ ] Screen S002 - 근거 상세

**Purpose:** 영향도 판정의 이유를 데이터 수치와 매크로 해석으로 보여준다. 뉴스 제목이 아니라 판단 근거를 보게 하는 화면.

**User Moment:** 플로우 1 단계 2 — "왜 위험/중립/호재인지 근거 펼치기", 플로우 2 단계 3 — "바뀐 근거 확인", 플로우 3 단계 3 — "다음 날 영향 이벤트 확인"

**Primary Action:** [내 보유 영향 보기] 선택

**Leads To:** S004

**States to Mock:** 기본(근거 있음), 장중 업데이트(바뀐 근거 강조), 데이터 지연(stale data 경고)

**Review Focus:**
- [ ] 지표 수치와 해석 문장의 정보 위계가 명확한가?
- [ ] "바뀐 근거"와 "기존 근거"를 구분할 수 있는가?

**Done When:**
- [ ] 주요 지표(금리·환율·지수·수급) 테이블 레이아웃이 표현됨
- [ ] 해석 문단이 표 아래에 위치하는 구조가 표현됨
- [ ] [내 보유 영향 보기] CTA가 명확함

## [ ] Screen S003 - 내 포트폴리오 입력

**Purpose:** 개인화된 민감도 진단에 필요한 최소 정보(종목, 현금 비중, 레버리지 여부)를 직접 입력한다. 로그인·계좌 연결 없이 시작.

**User Moment:** 플로우 1 단계 3 — "보유 종목과 현금 비중 입력 또는 수정"

**Primary Action:** [민감도 보기] 선택

**Leads To:** S004

**States to Mock:** 최초 입력(빈 상태), 수정 상태(기존 값 있음), 입력 오류(잘못된 값)

**Review Focus:**
- [ ] 입력 항목이 3개(종목·현금·레버리지)로 충분히 간결한가?
- [ ] 종목 추가·삭제 UX가 직관적인가?

**Done When:**
- [ ] 종목 리스트 + 추가/삭제 영역이 표현됨
- [ ] 현금 비중 슬라이더 또는 입력 필드가 표현됨
- [ ] 레버리지 여부 토글/라디오가 표현됨
- [ ] 빈 상태(최초 진입)와 수정 상태가 구분됨

## [ ] Screen S004 - 민감도 진단

**Purpose:** 내 포트폴리오가 오늘 시장 위험에 어떻게 노출됐는지 섹터·종목·레버리지 기준으로 보여준다.

**User Moment:** 플로우 1 단계 4 — "내 포트폴리오에 민감한 리스크 확인", 플로우 2 단계 4 — "내 보유 종목의 위험 노출 확인"

**Primary Action:** [행동 가이드 보기] 선택

**Leads To:** S005

**States to Mock:** 기본(위험 노출 있음), 낮음(노출 적음), 포트폴리오 미입력(일반 시장 경고만 표시)

**Review Focus:**
- [ ] 섹터별 위험 노출이 한눈에 파악되는가?
- [ ] 개인화 경고 문장이 시장 일반 경고와 구분되는가?

**Done When:**
- [ ] 섹터 노출 바 또는 리스트가 표현됨
- [ ] 주요 경고 문장 카드가 표현됨
- [ ] [행동 가이드 보기] CTA가 명확함

## [ ] Screen S005 - 행동 가이드

**Purpose:** 시장 판단과 포트폴리오 진단을 사용자의 실제 행동 기준으로 변환한다. 방어 문장(심리 코칭)이 핵심.

**User Moment:** 플로우 1 단계 5 — "오늘의 매수/보유/방어 기준 저장", 플로우 2 단계 5 — "팔기·버티기·축소 중 선택", 플로우 3 단계 4 — "내일 기준 결정 저장"

**Primary Action:** [오늘 기준 저장] 또는 [일부 축소] 선택

**Leads To:** S001 (저장 후 복귀), S008 (마감 전 흐름에서 알림 설정)

**States to Mock:** 아침 기준, 장중 보정 기준(판단 변경 후), 마감 전 내일 기준, 저장 완료(성공 피드백)

**Review Focus:**
- [ ] 매수/보유/레버리지 세 항목의 기준이 명확히 분리되는가?
- [ ] 방어 문장이 결정 전 질문 역할을 하는가?
- [ ] "오늘 기준 유지"와 "일부 축소" 버튼 선택이 명확한가?

**Done When:**
- [ ] 행동 기준 3개 항목(매수/보유/레버리지) 레이아웃이 표현됨
- [ ] 방어 문장 카드(심리 코칭 블록)가 표현됨
- [ ] 저장 완료 상태(성공 피드백)가 표현됨

## [ ] Screen S006 - 장중 변화 알림

**Purpose:** 아침 판단을 바꿀 만한 변화가 생겼을 때만 선별 호출한다. 기존 판단과 현재 판단을 나란히 보여줘 충동 매매 전 재점검을 유도.

**User Moment:** 플로우 2 단계 1 — "중요한 변화 알림 확인"

**Primary Action:** [무엇이 바뀌었나 보기] 선택

**Leads To:** S002 (근거 상세), S001 (닫기)

**States to Mock:** 위험 상향(중립→위험), 위험 하향(위험→중립), 무시(닫기)

**Review Focus:**
- [ ] "기존 판단 → 현재 판단" 비교가 즉시 눈에 들어오는가?
- [ ] 알림의 긴급도가 공포를 키우지 않으면서 전달되는가?

**Done When:**
- [ ] 기존/현재 판단 비교 레이아웃이 표현됨
- [ ] 변화 이유 요약 문장이 표현됨
- [ ] [무엇이 바뀌었나 보기] / [닫기] 두 버튼이 표현됨

## [ ] Screen S007 - 마감 전 점검

**Review Focus:**
- [ ] 오늘 요약과 내일 예정 이벤트가 명확히 분리되는가?
- [ ] 내일 기준 3개 항목(현금/매수/레버리지)이 간결하게 보이는가?

**Done When:**
- [ ] 오늘 요약 + 내일 이벤트 리스트 레이아웃이 표현됨
- [ ] 내일 기준 미리보기 블록이 표현됨
- [ ] [기준 저장] [아침 알림 켜기] 두 CTA가 표현됨

## Open Questions

- S001의 시간대 탭은 자동 선택(현재 시간 기준)이 기본인가, 사용자가 수동으로 전환하는가?
- S003 종목 입력 방식: 자유 텍스트(종목명 직접 입력)인가, 티커 검색 자동완성인가?
- S005 "오늘 기준 저장"은 로컬 저장인가, 서버 저장인가? v1에서 계정 없이 저장 가능한 범위를 결정해야 함.
- S006 장중 변화 알림은 알림 배너(OS 푸시)인가, 앱 내 상단 배너인가? v1 기준 명확화 필요.
- S007에서 내일 기준 미리보기와 S005 행동 가이드가 중복되는 영역이 있음 — 통합할지 분리할지 리뷰 시 결정.
