# Mockup Screens

## Overview

마켓가드 8개 화면의 상세 설계. 각 화면의 정보 구조, 상태, 리뷰 기준을 담는다.
`plan.md`의 Screen ID와 1:1 대응한다. ASCII 도식은 모바일(375px) 기준이다.

## Global Mockup Assumptions

- 레이아웃: 모바일 단일 컬럼, 상단 고정 헤더(마켓가드 워드마크 + 우상단 포트폴리오 아이콘), 하단 Safe Area 여백 확보
- 더미 데이터: 영향도 = "위험", 금리 +0.12bp, 환율 1,384원, 나스닥 -1.4%, 반도체 -2.1%, 보유 종목 = 삼성전자·SK하이닉스, 현금 비중 20%, 레버리지 있음
- 반응형 기준: 375px(모바일 기본) / 768px(태블릿) / 1280px(데스크톱). 데스크톱에서는 최대 너비 480px 카드를 중앙 정렬하거나 2컬럼 레이아웃으로 확장
- 색상 의미: 위험 = `#fe493d` (Error Red), 주의 = `#fd8700` (Warning Orange), 우호 = `#04c584` (Guard Mint), 중립 = `#7b7b7b`
- 카드 스타일: 2px radius, `0 2px 5px rgba(0,0,0,.12)` shadow, `#e1e1e1` border
- 타이포: Pretendard 700 for 결론·숫자·CTA, 500 for 본문·설명

## Screen S001 - 오늘의 방어 브리핑

**Purpose:** 사용자가 앱을 열자마자 오늘 한국장 영향도와 핵심 근거 3개를 확인하고 다음 행동을 선택한다. 서비스의 첫 인상이자 핵심 가치 전달 화면.

**Entry Points:**
- 앱 직접 진입 (북마크, 홈화면 아이콘)
- S005 [오늘 기준 저장] 후 복귀
- S006 [닫기] 후 복귀
- S008 [알림 켜기] 후 복귀

**Exit Points:**
- [근거 보기] → S002
- [내 포트 점검] → S003
- 마감 전 탭 전환 → S007

**Primary User Action:** 영향도 확인 후 [근거 보기] 또는 [내 포트 점검] 선택

**Secondary Actions:**
- 시간대 탭 수동 전환 (아침 / 장중 / 마감 전)
- 영향도 카드 전체 탭 → S002로 이동

**Layout ASCII:**

```text
┌────────────────────────────────────┐
│ 마켓가드              [포트폴리오⚙] │  ← 고정 헤더
├────────────────────────────────────┤
│  [아침 ●]   [장중]   [마감 전]      │  ← 시간대 탭 (현재 탭 밑줄+강조)
├────────────────────────────────────┤
│                                    │
│  오늘 한국장 영향도       08:42 기준│
│ ┌──────────────────────────────┐   │
│ │                              │   │
│ │         위  험               │   │  ← 영향도 뱃지 (대형, Error Red)
│ │                              │   │
│ └──────────────────────────────┘   │
│                                    │
│  금리 상승 + 반도체 약세 +          │
│  환율 부담이 겹쳤습니다.            │  ← 근거 요약 문장
│                                    │
│  지금 볼 것                         │
│  ① 반도체 선물 흐름                 │
│  ② 원/달러 환율                     │  ← 핵심 3개 항목
│  ③ 외국인 수급                      │
│                                    │
│ [근거 보기]      [내 포트 점검]     │  ← Primary CTA 2개
│                                    │
└────────────────────────────────────┘
```

**Key UI Regions:**
- Header: 서비스 워드마크(좌) + 포트폴리오 설정 아이콘(우)
- Tab Bar: 아침/장중/마감 전 탭. 현재 시간대 자동 선택, 수동 전환 가능
- Impact Card: 한국장 영향도 뱃지(대형 텍스트) + 갱신 시각
- Summary Text: 영향도 근거 1~2줄 요약
- Watch List: 지금 볼 것 3개 항목 번호 리스트
- CTA Row: [근거 보기](ghost) + [내 포트 점검](primary)

**Content / Copy Notes:**
- 영향도: "위험" / "중립" / "우호" — 세 단어만 사용
- 근거 요약: 1~2줄, 지표 + 해석 연결 ("금리 상승 + 반도체 약세 + 환율 부담이 겹쳤습니다")
- 갱신 시각: "08:42 기준" 형식, 지연 시 "10:12 이후 갱신 없음" 표시
- CTA: [근거 보기] [내 포트 점검] — 두 행동 모두 강요 없이 병렬 제시

**Data / Placeholder Needs:**
- 더미 영향도: 위험 (붉은 배경 또는 Error Red 텍스트)
- 더미 근거 요약: "금리 상승 + 반도체 약세 + 환율 부담이 겹쳤습니다."
- 더미 Watch List: ① 반도체 선물 흐름 ② 원/달러 환율 ③ 외국인 수급

**States:**
- Default: 위험 — Impact Card에 Error Red 색상, Watch List 표시
- 중립: Warning 표시 없이 중립 색상(`#7b7b7b`), 근거 문장 톤 완화
- 우호: Guard Mint(`#04c584`) 강조, "추격 매수 제한" 멘트 병기
- Loading: 영향도 카드 자리에 `#f5f5f5` 스켈레톤 블록, Watch List 자리에 3개 라인 스켈레톤. 숫자 슬롯은 `--` 표시
- Error(데이터 실패): Impact Card 자리에 `#fe493d` left-border 인라인 배너 "데이터를 불러오지 못했습니다. 잠시 후 다시 확인해 주세요." + [다시 확인] ghost 버튼
- 포트폴리오 미입력: 기본 브리핑은 정상 표시. [내 포트 점검] 버튼 아래 caption: "보유 종목을 추가하면 오늘 민감한 섹터를 함께 확인할 수 있습니다."

**Responsive Notes:**
- Mobile (375px): 단일 컬럼, Impact Card 전체 너비, CTA 버튼 2개 나란히
- Desktop (1280px): 카드 최대 너비 480px 중앙 정렬. 탭 바는 카드 상단에 고정

**Review Questions:**
- [ ] 영향도 뱃지(위험/중립/우호)가 첫눈에 들어오는가?
- [ ] 시간대 탭 전환 의도가 직관적으로 전달되는가?
- [ ] 포트폴리오 미입력 상태에서도 서비스 가치가 느껴지는가?

**Acceptance Criteria:**
- [ ] 위험/중립/우호 3가지 영향도 상태가 색상으로 구분됨
- [ ] 시간대 탭 3개가 표현되고 현재 탭이 강조됨
- [ ] 로딩·오류·포트폴리오 미입력 상태가 각각 표현됨
- [ ] [근거 보기]와 [내 포트 점검] 두 CTA가 명확히 표현됨

**Notes:**
시간대 탭은 현재 시간을 기준으로 자동 선택되지만, 사용자가 언제든 수동 전환 가능. 마감 전 탭 선택 시 S007로 이동하는 흐름이 plan.md와 일치해야 함.

## Screen S002 - 근거 상세

**Purpose:** 영향도 판정의 이유를 주요 지표 수치와 매크로 해석 문장으로 구체화한다. 숫자와 해석이 함께 있어 뉴스 제목이 아닌 판단 근거를 보게 한다.

**Entry Points:**
- S001 [근거 보기]
- S006 [무엇이 바뀌었나 보기]
- S004 [내 보유 영향 보기] 역방향 (브레드크럼 뒤로 가기)

**Exit Points:**
- [내 보유 영향 보기] → S004
- 상단 ← 뒤로가기 → S001

**Primary User Action:** [내 보유 영향 보기] 선택

**Secondary Actions:**
- 개별 지표 행 탭 → 해당 지표 상세 툴팁 (v1 선택사항)
- 뒤로가기 → S001

**Layout ASCII:**

```text
┌────────────────────────────────────┐
│ ←          근거 상세               │  ← 헤더 (뒤로가기 + 제목)
├────────────────────────────────────┤
│ 영향도: 위험          08:42 기준   │
├────────────────────────────────────┤
│ 지표              수치   방향  영향 │
│ ─────────────────────────────────  │
│ 금리 (10Y)       +0.12bp  ↑  부담  │
│ 원/달러 환율      1,384원  ↑  부담  │
│ 나스닥           -1.4%   ↓  약세   │
│ 반도체 지수(SOX)  -2.1%   ↓  약세  │
│ 외국인 수급       매도 기조   ↓ 약세│
├────────────────────────────────────┤
│ 해석                               │
│                                    │
│ 신호가 복수로 겹쳐 단기 소음보다    │
│ 위험 쪽에 무게가 실립니다.          │
│ 환율과 금리가 동시에 오른 날은      │
│ 반도체 중심 하락이 깊어질 수 있습니다.│
├────────────────────────────────────┤
│ [내 보유 영향 보기]                 │
└────────────────────────────────────┘
```

**Key UI Regions:**
- Sub-header: 현재 영향도 요약 뱃지 + 갱신 시각
- Data Table: 지표명 / 수치 / 방향 아이콘(↑↓) / 영향 라벨(부담/약세/호재). 값은 오른쪽 정렬 700weight, 위험 값은 Error Red
- Interpretation Block: 해석 문단 카드 (border-left 강조)
- CTA: [내 보유 영향 보기] primary 버튼

**Content / Copy Notes:**
- 지표명: 한국어 표기 우선 (금리, 환율, 나스닥 등 친숙한 명칭)
- 방향 아이콘: ↑(상승/부담) / ↓(하락/약세) — 색상으로 의미 강화
- 해석 문단: 2~3문장. "~입니다" 존댓말, 결론 → 이유 순서
- 데이터 지연 시: 각 수치 옆에 "지연" caption 표시

**Data / Placeholder Needs:**
- 더미 지표: 금리 +0.12bp / 환율 1,384원 / 나스닥 -1.4% / SOX -2.1% / 외국인 매도
- 더미 해석: "신호가 복수로 겹쳐 단기 소음보다 위험 쪽에 무게가 실립니다."

**States:**
- Default: 기본 근거 테이블 + 해석 문단
- 장중 업데이트: 변경된 지표 행에 "변경됨" 배지 또는 배경 강조 (`#f3fdfa` tint)
- Stale Data: 각 수치 뒤에 "10:12 이후 갱신 없음" caption, 비방해적 경고 배너

**Responsive Notes:**
- Mobile: 테이블 컬럼 4개 (지표/수치/방향/영향), 가로 스크롤 없이 고정
- Desktop: 테이블 좌우 여백 확보, 해석 블록과 나란히 2컬럼 가능

**Review Questions:**
- [ ] 지표 수치와 해석 문장의 정보 위계가 명확한가?
- [ ] 장중 업데이트 시 "바뀐 근거"가 한눈에 구분되는가?

**Acceptance Criteria:**
- [ ] 주요 지표 5개 이상의 테이블 레이아웃이 표현됨
- [ ] 해석 문단 카드가 테이블 아래에 위치함
- [ ] [내 보유 영향 보기] CTA가 명확함
- [ ] Stale data 경고 상태가 표현됨

**Notes:**
없음

## Screen S003 - 내 포트폴리오 입력

**Purpose:** 개인화 민감도 진단에 필요한 최소 3가지 정보(보유 종목, 현금 비중, 레버리지 여부)를 로그인 없이 직접 입력한다.

**Entry Points:**
- S001 [내 포트 점검]
- S001 헤더 포트폴리오 아이콘

**Exit Points:**
- [민감도 보기] → S004
- ← 뒤로가기 → S001

**Primary User Action:** 종목 입력 후 [민감도 보기] 선택

**Secondary Actions:**
- 종목 추가 [+ 추가]
- 종목 삭제 [×]
- 현금 비중 슬라이더 조절
- 레버리지 여부 토글

**Layout ASCII:**

```text
┌────────────────────────────────────┐
│ ←          내 포트폴리오            │
├────────────────────────────────────┤
│ 보유 종목                           │
│ ┌───────────────────┐ [+ 추가]     │
│ │ 삼성전자         × │             │
│ └───────────────────┘             │
│ ┌───────────────────┐             │
│ │ SK하이닉스       × │             │
│ └───────────────────┘             │
│                                    │
│ 현금 비중                           │
│ 0%  [────────○──────────]  100%   │
│                   20%              │
│                                    │
│ 레버리지                            │
│ ○ 없음   ● 있음                    │
├────────────────────────────────────┤
│ 보유 종목을 추가하면 오늘 민감한     │
│ 섹터를 함께 확인할 수 있습니다.     │  ← caption (빈 상태)
├────────────────────────────────────┤
│ [민감도 보기]                       │
└────────────────────────────────────┘
```

**Key UI Regions:**
- 보유 종목: 종목 칩 리스트 + [+ 추가] 버튼. 텍스트 입력 또는 자동완성 (v1: 자유 텍스트)
- 현금 비중: 슬라이더 또는 숫자 직접 입력 필드 (Amount Input 스타일, 우측 정렬 700weight)
- 레버리지: 라디오 버튼 2개 (없음 / 있음)
- Empty caption: 종목 미입력 시 안내 문구
- CTA: [민감도 보기] primary 버튼 (종목 1개 이상 입력 시 활성화)

**Content / Copy Notes:**
- 종목 입력 placeholder: "종목명 입력 (예: 삼성전자)"
- 현금 비중 label: "현금 비중" + 슬라이더 우측에 퍼센트 값 실시간 표시
- 레버리지 label: "레버리지 ETF 또는 2배 이상 상품 보유 여부"
- Empty 안내: "보유 종목을 추가하면 오늘 민감한 섹터를 함께 확인할 수 있습니다."

**Data / Placeholder Needs:**
- 더미 종목: 삼성전자, SK하이닉스
- 더미 현금 비중: 20%
- 더미 레버리지: 있음

**States:**
- Default(수정): 기존 값 표시 — 종목 칩, 슬라이더 위치, 레버리지 선택
- Empty(최초 진입): 종목 리스트 비어있음. Caption 표시. [민감도 보기] 비활성화
- Error(잘못된 입력): 종목명 필드 아래 인라인 오류 메시지 (예: "인식할 수 없는 종목명입니다")

**Responsive Notes:**
- Mobile: 전체 너비 종목 칩, 슬라이더 전체 너비
- Desktop: 입력 필드 최대 너비 480px

**Review Questions:**
- [ ] 3개 입력 항목이 한 화면에서 부담 없이 파악되는가?
- [ ] 종목 추가·삭제 UX가 직관적인가?

**Acceptance Criteria:**
- [ ] 종목 칩 리스트 + [+ 추가] / [×] 삭제 영역이 표현됨
- [ ] 현금 비중 슬라이더와 수치가 표현됨
- [ ] 레버리지 여부 선택 UI가 표현됨
- [ ] 빈 상태(최초 진입)와 수정 상태가 구분됨

**Notes:**
v1에서 종목 입력은 자유 텍스트. 자동완성·티커 검색은 Open Question으로 남긴다.

## Screen S004 - 민감도 진단

**Purpose:** 내 포트폴리오가 오늘 시장 위험에 어떻게 노출됐는지 섹터·종목·레버리지 기준으로 시각화한다. 개인화된 경고가 일반 시장 경고와 구분된다.

**Entry Points:**
- S003 [민감도 보기]
- S002 [내 보유 영향 보기]

**Exit Points:**
- [행동 가이드 보기] → S005
- ← 뒤로가기 → S002 또는 S003

**Primary User Action:** [행동 가이드 보기] 선택

**Secondary Actions:**
- 개별 섹터 행 탭 → 해당 섹터 상세 (v1 선택사항)

**Layout ASCII:**

**Key UI Regions:**
- Preview Text: 알림이 어떤 내용을 보낼지 미리보기 ("오전 8:40에 한국장 영향도를 알려드릴게요")
- Time Chips: 4개 시간 선택 칩 (8:00 / 8:30 / 8:40 기본 / 9:00)
- CTA: [알림 켜기] primary 버튼

**Content / Copy Notes:**
- Preview: "오전 {시간}에 '오늘 한국장 영향도'를 알려드릴게요." 형식
- 알림 설명: "판단이 필요한 변화가 생기면 추가로 알려드립니다." caption
- 완료 피드백: "알림이 켜졌습니다. 내일 {시간}에 확인하세요."

**Data / Placeholder Needs:**

- 더미 완료 메시지: "알림이 켜졌습니다. 내일 8:40에 확인하세요."

**States:**
