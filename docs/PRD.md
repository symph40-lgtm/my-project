## 1. 개요

- **한 줄 정의:** 변동성 장에서 감정적 매매를 반복하는 직장인 개인투자자에게, 미국장·매크로·수급·포트폴리오 데이터를 근거로 시간대별(아침·점심·장마감 전) 행동 판단을 제공하는 방어형 시장 코치 서비스.
- **해결하는 문제:** 급락장에서 공포로 시장을 외면하거나 반등에 휩쓸려 추격매수하는 행동. 시장 데이터를 각자 수집·해석해야 해서 시간이 많이 들고, 데이터와 해석이 분리되어 있는 문제.
- **타깃 사용자:** 출근 전·점심시간·장마감 전에 시장을 확인하는 직장인 개인투자자. 주식 경험은 초보~중급, 변동성 장에서 감정적으로 흔들리기 쉬움. 한국 주식(코스피·코스닥) 투자 중심.
- **플랫폼:** 반응형 웹 (모바일 우선, 데스크톱 지원)

---

## 2. 배경 & 근거

- **시장/경쟁 맥락:** Bloomberg Terminal·FactSet은 기관 투자자 대상으로 가격·복잡도 장벽이 높다. Koyfin·TradingView는 사용자가 직접 데이터를 해석해야 하며 초보에게 신호가 너무 많다. TipRanks·PortfolioPilot은 점수·AI 조언 방향은 가깝지만, 급락장 당일 한국 개인투자자 언어로 번역한 행동 코치에는 미치지 못한다.
- **우리의 차별 프레이밍:** 전문가용 데이터 깊이를 개인 투자자의 하루 판단 언어로 번역하는 방어형 시장 코치. 예측 수익률보다 `위험도 + 근거 + 행동 제한선`을 먼저 보여주고, 매수/매도 버튼을 누르기 전 "지금 결정이 손실 만회 심리인지, 데이터 근거인지"를 점검하게 만든다.

---

## 3. 목표 (Goals)

1. 직장인 투자자가 아침 9시 이전, 점심, 장마감 전에 5분 안에 오늘의 시장 위험도와 행동 방침을 파악할 수 있게 한다.
2. 변동성 장에서 추격매수·공포 회피·손실 만회 매매 빈도를 줄이는 행동 코치 역할을 한다.
3. 보유 포트폴리오(종목·섹터·레버리지·현금 비중)를 반영한 개인화된 위험 경고로 서비스 체감 가치를 높인다.

## 4. User Scenarios

### Scenario 1 — 아침 방어 브리핑 "장 시작 전에 오늘은 뭘 조심해야 하지?"

- **사용자:** 코스피·코스닥 주식을 보유한 직장인 투자자
- **상황:** 출근 전 오전 7~9시. 전날 미국장에서 큰 움직임이 있었고 오늘 장이 걱정됨. 막연한 불안 상태.
- **목표:** "오늘 무리해도 되는 날인지, 방어해야 하는 날인지" 판단 — 오늘의 시장 위험과 행동 기준을 아는 상태로 전환
- **흐름:** 앱 진입 → 오늘의 방어 브리핑(한국장 영향도 + 핵심 근거 3개) 확인 → 근거 상세에서 매크로 해석 확인 → 포트폴리오 입력 또는 수정 → 민감도 진단(내 종목·섹터 노출 확인) → 행동 가이드(매수/보유/방어 기준) 확인 후 오늘 기준 저장
- **완료 조건:** 사용자가 한국장 영향도와 오늘의 행동 기준(매수 조심 / 보유 유지 / 레버리지 금지 등)을 장 시작 전에 확인하고 저장한다.

### Scenario 2 — 장중 판단 보정 "지금 흔들리는데 팔아야 하나?"

- **사용자:** 아침 브리핑을 확인한 직장인 투자자
- **상황:** 장중 급락 또는 급반등, 이슈 업데이트 알림이 뜸. 충동적으로 팔고 싶거나 추가 매수 충동이 생김.
- **목표:** 충동 매매 전에 "바뀐 정보"와 "내 감정"을 구분하여 판단을 재점검
- **흐름:** 장중 변화 알림 확인(기존 판단 vs 현재 판단 비교) → 근거 상세에서 무엇이 바뀌었는지 확인(변동성 패턴 감지 포함) → 민감도 진단으로 내 보유 종목 영향 확인 → 행동 가이드에서 "지금 파는 이유가 데이터인가, 공포인가" 결정 전 질문 → [오늘 기준 유지] 또는 [일부 축소] 선택
- **완료 조건:** 사용자가 충동 매도 또는 추격 매수 전에 앱을 통해 판단 근거를 확인하고, 무리한 행동 없이 결정에 도달한다.

### Scenario 3 — 마감 전 리스크 정리 "내일은 어떻게 준비하지?"

- **사용자:** 하루 거래를 마무리하는 직장인 투자자 (특히 레버리지·변동성 종목 보유자)
- **상황:** 장 마감 전. 오늘 하락의 원인이 내일까지 이어질지 불확실함.
- **목표:** 내일의 리스크를 예상하고 현금·보유·레버리지 기준을 미리 결정 — "하루 손익에 끌려가지 않는 상태"로 전환
- **흐름:** 마감 전 점검 화면(오늘 요약 + 내일 확인할 이벤트) → 민감도 진단으로 내일 종목·섹터 노출 확인 → 근거 상세에서 다음 날 영향 이벤트(CPI·고용·금리·환율) 확인 → 행동 가이드에서 내일 기준(현금 비중·레버리지 금지) 저장 → 내일 아침 알림 설정
- **완료 조건:** 사용자가 장 마감 전에 다음 거래일 행동 기준을 저장하고 아침 알림을 설정한다.

## 6. Success Criteria

- SC-001: 사용자가 아침 브리핑 화면 진입 후 한국장 영향도와 핵심 행동 방침을 3분 안에 확인할 수 있다.
- SC-002: 위험 구간 브리핑에서 사용자가 행동 힌트(레버리지 축소, 현금화 등)를 확인하고 즉각적인 충동 매수를 제한하는 판단에 도달할 수 있다.
- SC-003: 포트폴리오를 입력한 사용자가 자신의 보유 종목·섹터 기준 개인화된 경고를 받는다. *(가설 — userflow.md 및 실사용 검증 후 기준 구체화 필요)*
- SC-004: 서비스가 3개 시간대(아침/점심/장마감 전) 모두에서 해당 시간에 브리핑을 제공한다.

## 7. Edge Cases

- **포트폴리오 미입력(빈 상태):** 일반 시장 브리핑만 표시. 포트폴리오 추가 시 개인화된 경고를 받을 수 있다는 안내 메시지 표시. 입력 강요 없이 서비스 이용 가능.
- **외부 데이터 fetch 실패(미국장·환율·금리 데이터 수집 불가):** 오류 사실과 마지막 갱신 시간을 브리핑 카드 내 인라인으로 표시. 이전 수치는 "지연 데이터" 표시와 함께 유지. 전체 화면 오류 대신 해당 카드만 오류 처리.
- **데이터 지연(stale data):** 갱신 시간을 항상 표시. 갱신이 N분 이상 지연되면 비방해적 경고 배너 표시.
- **장 휴장일·주말:** 당일 브리핑 대신 "오늘은 시장이 열리지 않습니다" 안내와 다음 거래일 예정 브리핑 시간 표시.
- **시장이 '우호' 판단이지만 브리핑 중 급등 종목 추격 충동:** 우호 브리핑에도 추격매수 제한 멘트를 병기("우호 신호가 우세하지만, 급등 종목 추격 매수는 제한하세요").
- **위험 신호 복수 겹침:** 알림 강도를 자동으로 '비상'으로 올리고 가장 즉각적인 방어 행동(레버리지 축소·현금화)을 우선 제시.
- **포트폴리오 입력값 오류(형식 불일치·누락):** 필드별 인라인 오류 메시지. 불완전 입력 상태에서도 일반 브리핑은 정상 제공.
- **사용자가 알림을 무시하고 이탈 후 재진입:** 다음 시간대 브리핑에서 이전 판단 이후 시장 변화 요약을 첫 화면에 표시.
- **중복 알림 구독 시도:** 이미 구독 중임을 안내하고 구독 설정 변경 옵션 제공.

  - Do: 결론 먼저(`위험 구간입니다`), 이유와 행동을 뒤에 붙임

  - Don't: `대박`, `지금 안 사면 늦습니다`, `AI가 추천합니다` 금지
  - Don't: `괜찮을 수도 있어요` 같은 모호한 안심 금지
  - Don't: 느낌표, 이모지 금지. 긴급함은 정보 구조로 전달
- **시각 디자인 시스템:** → [docs/DESIGN.md](./DESIGN.md) 참조 (Guard Mint `#04c584` / Pretendard 700 / 2px radius / 시간대별 브리핑 카드 컴포넌트 포함)

## 10. 범위 / 비범위 & 우선순위

- **v1 범위 (P0 + 핵심 P1):**
  - 3개 시간대 시장 브리핑(아침·점심·장마감 전)
  - 한국장 영향도(위험/중립/우호) + 근거 2~4개
  - 시간대별 행동 힌트 + 알림 강도(주의/경고/비상)
  - 이슈 지속 가능성 판단
  - 포트폴리오 입력 및 개인화 경고
  - 오전장 수급 요약과 아침 판단 차이 표시(점심 브리핑)
  - 변동성 장 패턴 감지
  - 원칙 코칭 멘트 + 심리 방어 문장
  - 장마감 전 다음날 갭하락 가능성 포함 결론

- **Out of scope (지금 안 함):**
  - 종목 직접 매수·매도 실행 기능 (투자 실행은 서비스 범위 밖)
  - 미국 주식 직접 투자 포트폴리오 지원 (v1은 한국 시장 중심)
  - 로그인·계좌 연결 (v1은 직접 입력으로 시작)
  - 카카오톡·이메일 알림 (웹 푸시는 P2 검토)
  - 과거 유사 구간 차트 아카이브 (근거 상세는 현재 데이터 수준)
  - "고수의 전략" 별도 콘텐츠 화면 (보조 문장 수준은 P2에서 검토)
  - 소셜 기능(브리핑 공유·커뮤니티)
  - 자동 포트폴리오 연동(증권사 API 연결)

- **우선순위 묶음(제품 관점):**
  - MVP (P0): 시간대별 브리핑 + 한국장 영향도 + 근거 + 행동 힌트 + 알림 강도
  - v1 (P0 + 핵심 P1): MVP + 포트폴리오 개인화 + 수급 요약 + 패턴 감지 + 원칙 코칭 + 장마감 결론 강화
  - 이후 (P2+): 웹 푸시 알림, 근거 아카이브, 고수의 전략, 자동 포트폴리오 연동

- *구현 마일스톤 로드맵은 vibe-build의 roadmap 스킬이 docs/ROADMAP.md에 작성한다.*

## 11. Assumptions

1. 데이터 수집은 외부 금융 API(미국 주요 지수, 금리, 환율, 유가, 수급 데이터 등)를 통해 자동화된다. 데이터 소스·비용은 구현 전 별도 검토한다.
2. 브리핑 텍스트는 AI/LLM이 수집 데이터를 해석해 자연어 한국어 문장으로 생성한다.
3. v1에서는 한국 주식 시장(코스피·코스닥) 중심이며, 미국 직접 투자 포트폴리오는 범위 밖이다.
4. v1에서 로그인과 증권사 계좌 연결은 필수 흐름에서 제외한다. 포트폴리오는 직접 입력(종목명, 현금 비중, 레버리지 여부)으로 시작한다.
5. 포트폴리오 입력 범위는 v1 기준 종목, 현금 비중, 레버리지 여부까지만이다. 섹터 입력은 종목에서 자동 분류를 가정한다.
6. 아침·점심·장마감 전 브리핑은 사전 정의된 시간 기준으로 서버에서 자동 생성되는 방식이다.
7. 매수/매도 권고는 직접 명령이 아닌 행동 가이드와 방어 문장 중심으로 표현한다. 투자 실행 기능은 제공하지 않는다.
8. 매매 원칙(추격매수 금지 등)은 서비스가 미리 정의한 고정 규칙셋을 기반으로 코칭 멘트를 생성한다.
9. v1에서는 알림을 모든 뉴스가 아닌 판단 변경 가능성이 있는 변화에만 전송한다. 웹 푸시는 P2.

## 12. 오픈 이슈 / 리스크

- **[ ] 데이터 소스 미결정:** 미국장·금리·환율·유가·수급 데이터를 어느 API에서 얼마에 수집할지 미정. 비용 구조와 데이터 신뢰성이 서비스 핵심 품질에 직결됨.
- **[ ] AI/LLM 브리핑 생성 품질 검증 필요:** LLM이 생성한 해석 문장의 정확도와 일관성 기준 미정. 오판 시 사용자 손실로 이어질 수 있는 서비스 특성상 품질 기준과 면책 범위 사전 설계 필요.
- **[ ] 투자 조언 법적 이슈 검토 필요:** 행동 힌트가 투자 권유로 해석될 가능성. 금융투자업법상 면책 고지 범위와 표현 방식 법률 검토 필요.
- **[ ] 포트폴리오 입력 방식 미확정:** idea.md에서 종목·섹터·레버리지·현금 비중 입력을 언급했으나, 구체적인 입력 UX(직접 입력, 증권사 API 연동, 티커 검색 등) 미결정.
- **[ ] 알림 방식 미확정:** 웹 푸시, 카카오톡, 이메일 중 v1에서 지원할 채널 미결정. Assumptions에서 웹 푸시만 P2로 가정했으나 확정 필요.
- **[ ] 한국장 영향도 판정 알고리즘 미정:** 위험/중립/우호 결론을 어떤 데이터 조합과 가중치로 판정할지 기준 미정. 이 알고리즘이 서비스의 신뢰도 핵심.
